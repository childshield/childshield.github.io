Security

Factors
The security of an IoToy can be determined by observing the following factors:
  • S1: Personal Information (PI) Storage Security
  • S2: Communication Security
  • S3: App Privileges
  • S4: Code Security
  • S5: Device Security
  • S6: Maintenance

Rubrics

Security Factor Score (xi)
-1 -0.5 0 0.5 1
S1 No security measures are applied on PI storage PI storage security measures exist but are weak and unsatisfactory to ensure confidentiality, integrity and availability Not applicable PI storage security measures can be improved to ensure confidentiality, integrity and availability Confidentiality, integrity and availability of PI is guaranteed. Sensitive PI is encrypted using strong methods and accessible by authenticated and authorised users only
S2 PI is transmitted through insecure channels Weak communication protocols are used. For example, insecure versions of SLS/TLS, weak key lengths or default passwords for pairing Not applicable There is scope for improving communication security Mutual authentication and encryption practices are strong
S3 Companion desktop or mobile app is over privileged. User is not notified of app privileges during installation Companion desktop or mobile app has few unnecessary privileges. User is not notified of app privileges during installation Not applicable Companion desktop or mobile app has few unnecessary privileges. User is notified of app privileges during installation Companion desktop or mobile app uses only necessary privileges. User is notified of app privileges during installation
S4 Secure coding practices are not followed resulting in insecure code Few secure coding practices are followed and many vulnerabilities are detected in source code Not applicable Most secure coding practices are followed and few vulnerabilities are detected in source code Code demonstrates secure coding practices
S5 Device (physical toy components) is insecure and exhibits vulnerabilities that may have large scale impact or affect the immediate user's safety Device (physical toy components) exhibits vulnerabilities that may have minor impact to the immediate user Not applicable Device (physical toy components) provides weak credentials, password management or encryption methods Device (physical toy components) is secure
S6 Either of the following is true:
  • No patches/ updates are provided for bugs and security vulnerabilities
  • Manufacturer has not provided a maintenance duration
 Few updates/patches are provided for bugs and security vulnerabilities during maintenance duration Not applicable Most updates/patches for bugs and security vulnerabilities are provided timely during maintenance duration Updates/patches are quickly made available for bug fixes and security vulnerabilities discovered during maintenance duration
